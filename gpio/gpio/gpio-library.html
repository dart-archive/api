GPIO support.

Provides access to controlling GPIO pins.

The library provide two ways of accessing the GPIO pins: direct access or access through a Sysfs driver.

When direct access is used, the physical memory addresses, where the SoC registers for the GPIO pins are mapped, are accessed directly. This always require root access.

When the Sysfs driver is used the filesystem under /sys/class/gpio is used; root access is also required by default. However this can be changed through udev rules, e.g. by adding a file to /etc/udev/rules.d. In addition the Sysfs driver supports state change notifications.

Currently this has only been tested with a Raspberry Pi 2.
